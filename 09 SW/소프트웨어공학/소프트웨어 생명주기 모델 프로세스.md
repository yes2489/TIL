# 소프트웨어 생명주기 모델 (Software Development Life Cycle)
> 소프트웨어를 개발하고 유지보수하기까지의 전체 생명주기를 단계별로 구분한 모델

**목적**
- 체계적인 개발 관리
- 품질 확보
- 일정·비용 통제

## 소프트웨어 생명주기 모델 프로세스
### 1. 요구사항 분석
> 사용자가 원하는 기능과 제약사항을 미리 파악하는 단계

**주요 내용**
- 요구사항 도출
- 요구사항 분석
- 요구사항 명세
- 요구사항 확정 (검증)

**요구사항 분류**
- 기능적 요구사항: 시스템이 **수행해야 할** 기능 (`주문 처리`, `로그인 기능`)
- 비기능적 요구사항: 성능, 보안, 신뢰성 등 **품질** 요소 (`동시에 200명 처리`, `응답시간 2초 이내`)

**산출물**
- 요구사항 명세서 (SRS)

### 2. 설계
> 요구사항을 만족하기 위한 시스템 구조를 정의하는 단계

**설계 단위 구분**
- 상위 설계 (아키텍처 설계)
    - 시스템 전체 구조
    - 모듈 간 관계
- 하위 설계 (상세 설계)
    - 모듈 내부 구조
    - 클래스, 인터페이스 설계

**설계 도구**
- UML 다이어그램
    - 클래스 다이어그램
    - 시퀀스 다이어그램

**산출물**
- 설계 명세서
- UML 설계 문서

### 3. 구현
> 설계 결과를 기반으로 실제 프로그램을 작성하는 단계

**주요 내용**
- 프로그래밍 언어를 사용한 코드 작성
- 모듈 단위 구현
- 인터페이스, 자료 구조 구현
- 예외 및 오류 처리

**산출물**
- 소스 코드
- 실행 파일

### 4. 테스트
> 구현된 소프트웨어가 요구사항을 만족하는지 검증하는 단계

**테스트 수준**
- 단위 테스트
- 통합 테스트
- 시스템 테스트
- 인수 테스트

**검증 관점**
- Verification: 설계대로 구현되었는가
- Validation: 사용자 요구를 만족하는가

**목적**
- 오류 발견 및 품질 확보

**산출물**
- 테스트 결과서
- 결함 보고서

### 5. 유지보수
> 운영중인 소프트웨어를 수정·개선하는 단계

**유지보수 유형**
- 수정 유지보수: 오류 수정
- 적응 유지보수: 환경 변화 대응
- 완전화 유지보수: 기능 개선
- 예방 유지보수: 미래 오류 예방
