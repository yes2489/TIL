# 3R

## 역공학 (Reverse Engineering)
> 기존에 존재하는 소프트웨어를 분석하여 상위 수준의 설계·구조·명세를 추출하는 기법

**목적**
- 소스 코드 이해
- 문서 부재 시스템 분석
- 유지보수 및 재공학을 위한 사전 작업

**특징**
- 기능은 변경하지 않음
- 구현 → 설계 방향으로 분석

**산출물**
- 설계 문서
- UML 다이어그램
- 구조/데이터 흐름 정보

## 재공학 (Re-engineering)
> 기존 소프트웨어를 분석 후 구조·성능·유지보수성을 개선하여 재구성하는 기법

**목적**
- 성능 개선
- 유지보수성 향상
- 기술 부채 해소

**특징**
- 기능은 유지, 구조는 개선
- 역공학 + 순공학 결합
- 내부 구조 변경 가능

**주요 활동**
- 코드 리팩터링
- 데이터 구조 개선
- 아키텍처 개선

## 재사용 (Reuse)
> 이미 개발된 소프트웨어 자산을 활용하여 새로운 시스템을 개발하는 기법

**목적**
- 개발 비용 절감
- 개발 기간 단축
- 품질 향상

**재사용 대상**
- 코드
- 컴포넌트
- 라이브러리
- 프레임워크
- 설계 패턴

**재사용 수준**
- 블록 재사용
- 컴포넌트 재사용
- 시스템 재사용
