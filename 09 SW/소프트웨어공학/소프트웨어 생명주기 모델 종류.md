# 소프트웨어 생명주기 모델

## 1. 폭포수 모델 (Waterfall Model)
> 이전 단계가 완료되어야 다음 단계로 진행하는 **순차적 모델**

**특징**
- 각 단계가 명확히 구분됨
    `타당성 검토 → 계획 → 요구사항 분석 → 설계 → 구현 → 테스트 → 유지보수`
- 문서 중심 개발
- 단계 간 되돌아가기 어려움

**장점**
- 관리가 용이함
- 요구사항이 명확한 프로젝트에 적합

**단점**
- 요구사항 변경에 취약
- 후반부에 오류 발견 가능성 큼

## 2. 프로토타이핑 모델 (Prototyping Model)
> 초기 주요 요구사항을 프로토타입으로 구현하여 사용자 피드백을 통해 요구사항을 구체화하는 모델

**특징**
- 사용자와의 상호작용 중심
- 요구사항을 반복적으로 보완

**장점**
- 요구사항 불명확성 감소
- 사용자 만족도 향상

**단점**
- 비용산정 어려움

## 3. 나선형 모델 (Spiral Model)
> 시스템 개발 시 위험 분석을 중심으로 점진적으로 시스템을 완성해 나가는 모델

**특징**
- 반복 개발
    `계획 → 위험 분석 → 개발 → 평가`
- 위험 분석을 핵심 활동으로 포함

**장점**
- 대규모·복잡한 시스템에 적합
- 위험 요소를 조기에 제거 가능

**단점**
- 관리 복잡
- 비용·기간 증가 가능성

## 4. 반복적 모델 (Iteration Model)
> 시스템을 실행 가능한 완성본 단위로 나누고, **각 증분마다 설계·구현·테스트 과정을 반복**하여 기능을 점진적으로 확장해 나가는 모델

**특징**
- 기능 단위 점진적 개발
- 각 반복마다 실행 가능한 결과물 생성

**장점**
- 변경 대응 용이
- 조기 피드백 가능

**단점**
- 전체 구조 설계 미흡 시 혼란 발생

## 5. V-모델 (V-Model)
> 개발 단계와 테스트 단계를 1:1로 대응시킨 모델

**특징**
- 각 개발 단계에 대응하는 테스트 단계 존재
- 검증과 확인 강조

**단계 대응**
- 요구사항 분석 ↔ 인수 테스트
- 설계 ↔ 시스템 테스트
- 상세 설계 ↔ 통합 테스트
- 구현 ↔ 단위 테스트

**장점**
- 테스트 계획 수립이 명확
- 품질 관리에 유리

**단점**
- 요구사항 변경에 취약
