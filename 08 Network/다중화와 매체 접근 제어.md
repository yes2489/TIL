# 다중화와 매체 접근 제어

## 다중화(Multiplexing)
 **하나의 전송 매체를 여러 장치가 동시에 사용**하기 위해 **데이터를 분할**하여 전송하는 기술

 ### FDM (Frequency Division Multiplexing)
 - 주파수 대역을 여러 개로 나누어 동시에 전송
 - 각 신호는 서로 다른 주파수 사용
 - `아날로그 통신`, `라디오 방송`

 ### TDM (Time Division Multiplexing)
 - 시간을 일정한 슬롯으로 나누어 순차적으로 전송
 - 각 장치는 할당된 시간에만 전송
 - 디지털 통신에서 주로 사용
 
 ### WDM (Wavelength Division Multiplexing)
 - 서로 다른 파장(빛의 색상)을 사용하여 동시에 전송
 - 광섬유 통신에서 사용
 - 대용량, 고속 전송 가능

## 공유 매체 접근 제어 (CSMA)
 **여러 장치가 하나의 전송 매체를 공유**할 때 **충돌을 제어**하기 위한 방식
 > **CSMA의 한계**  
 > 전파 지연으로 인해 다른 장치의 프레임이 아직 도달하지 않았을 경우,  
 > 매체를 사용 중이 아님으로 오인할 수 있다. (전파 지연)

 ### CSMA/CD (Collision Detection)
 - 전송 전에 매체 상태 확인
 - **충돌 발생 시 이를 감지하고 전송 중단 후 재전송**
 - 유선 이더넷 환경에서 사용

 ### CSMA/CA (Collision Avoidance)
 - 충돌을 사전에 회피하는 방식
 - **전송 전 랜덤 대기 시간 사용**
 - 무선 네트워크(Wi-Fi)에서 사용

## HDLC (High-Level Data Link Control)
 데이터 링크 계층에서 사용되는 비트 지향 프로토콜
 - 프레임 단위 통신
 - 오류 검출 및 흐름 제어 지원
 - 동기식 전송 방식 사용
 - 점대점 및 다중 접속 환경 지원

 ### 프레임 구조
 - **플래그(Flag)**: 프레임의 시작과 끝을 표시
 - **주소(Address)**: 데이터 수신자 주소
 - **제어(Control)**: 프레임 유형 식별 및 제어 정보 포함
 - **데이터(Data)**: 실제 전송되는 데이터
 - **FCS(Frame Check Sequence)**: 오류 검출을 위한 필드(CRC 사용)

 ### 프레임 유형
 - 정보 프레임 (I-Frame)
  - 구성: 주소, 제어, 데이터, FCS
  - 실제 사용자 데이터 전송에 사용
 - 감독 프레임 (S-Frame)
  - 구성: 주소, 제어, FCS
  - 흐름 제어 및 오류 제어 담당
 - 비번호 프레임 (U-Frame)
  - 구성: 주소, 제어, FCS
  - 링크 설정, 해제 등 제어 메시지 전송
