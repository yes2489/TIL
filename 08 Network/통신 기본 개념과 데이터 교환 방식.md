# 통신 기본 개념과 데이터 교환 방식

## 프로토콜 (Protocol)
 장비 간 데이터를 송수신하기 위한 메시지 양식과 규칙 체계

## 데이터 교환 방식
### 회선 교환
 통신 전에 송·수신자 간 전용 회선을 설정한 후 데이터를 전송하는 방식 `ex) 전화망`
 - 대역폭 독점
 - 지연이 일정함
 - 자원 활용 효율이 낮음

### 패킷 교환
 데이터를 패킷 단위로 분할하여 네트워크 자원을 공유하며 전송하는 방식 `ex) 인터넷`
 - 패킷별 경로가 달라질 수 있음
 - 순서 보장 없음
 - 지연 시간이 일정하지 않음
 - 자원 활용 효율 높음

## 데이터 전송 방향 기준 분류
### 단방향(Simplex)
 - 한 방향으로만 데이터 전송이 가능
 - `라디오`, `TV`

### 반이중(Half Duplex)
 - 양방향 전송이 가능하지만 **동시에 전송 불가**
 - `무전기`

### 전이중(Full Duplex)
 - 양방향 동시 전송 가능
 - `전화`, `이더넷 통신`

## 전송 동기화 방식
### 동기 전송 방식 (Synchronous)
 송신자와 수신자가 **공통된 시간 기준**을 사용하여 데이터를 연속적인 **프레임** 단위로 전송
 - 시작, 종료 비트 없음
 - 오버헤드가 적음
 - 전송 속도가 빠름
 - 프레임 경계를 기준으로 동기화

### 비동기 전송 방식 (Asynchronous)
 송신자와 수신자가 별도의 시간 기준 없이 **독립적**으로 **문자** 또는 **바이트** 단위로 데이터를 전송
 - 시작 비트와 종료 비트 포함
 - 오버헤드가 큼
 - 전송 속도가 느림
 - 문자 단위로 독립적인 동기화 수행

## 전송 형태
### 직렬 전송
 데이터를 한 번에 한 비트씩 순차적으로 전송
 - 구현이 단순
 - 장거리 전송에 적합

### 병렬 전송
 여러 비트를 동시에 전송
 - 속도가 빠름
 - 구현이 복잡
 - 근거리 전송에 적합
