# 전송 제어 매커니즘
> 종단 간 통신에서 데이터의 신뢰성을 보장하기 위한 기법   
> 주로 전송 계층(TCP)에서 수행

## 흐름 제어(Flow Control)
> 송신자가 수신자의 처리 능력을 초과하지 않도록 전송량을 조절하는 기법
 - 수신자의 버퍼 오버플로우 방지
 - 송신 속도 조절이 목적

 **대표 기법**
 - 정지-대기 방식
 - 슬라이딩 윈도우 방식

 ※ 흐름 제어는 **수신자의 처리 능력**을 고려하며, 네트워크의 혼잡 상태를 고려하는 **혼잡 제어**와는 구분됨

## 오류 제어(Error Control)
> 전송 중 발생한 **데이터 손실 또는 오류를 검출하고 복구**하는 기법
 - 오류 검출: 체크섬, CRC
 - 오류 복구: 재전송 요청

## ARQ(Automatic Repeat reQuest)
> 오류가 발생한 프레임 또는 패킷을 **자동으로 재전송**하는 방식

※ ARQ는 **오류 제어**를 구현하는 대표적인 기법

### Stop-and-Wait ARQ
 - 하나의 프레임 전송 후 ACK 수신 대기
 - ACK 수신 후 다음 프레임 전송

**특징**
 - 구조 단순
 - 전송 효율 낮음
 - 지연이 큰 환경에 부적합

### Go-Back-N ARQ
 - 여러 프레임을 연속으로 전송
 - 오류 발생 시 **오류 발생 지점 이후 프레임을 모두 재전송**

**특징**
 - 전송 효율 향상
 - 불필요한 재전송 발생 가능

### Selective Repeat ARQ
 - 여러 프레임을 연속으로 전송
 - 오류가 발생한 프레임만 **선택적으로 재전송**
 
**특징**
 - 전송 효율 가장 높음
 - 구현 복잡

## ARQ 방식 비교

| 구분 | Stop-and-Wait | Go-Back-N | Selective Repeat |
|----|----|----|----|
| 전송 방식 | 1개씩 | 연속 | 연속 |
| **재전송 범위** | 해당 프레임 | 이후 전체 | 오류 프레임만 |
| 효율 | 낮음 | 중간 | 높음 |
| 구현 난이도 | 낮음 | 중간 | 높음 |

## 패킷 스케줄링(Packet Scheduling)
> 여러 패킷 중 **어떤 패킷을 먼저 전송할지** 결정하는 기법
 - 네트워크 자원 효율적 사용
 - 지연 최소화
 - 서비스 품질(QoS) 보장 목적
 - 주로 **전송 계층 또는 네트워크 계층**에서 고려됨

**예시**
 - 우선순위 기반 전송
 - 공정성 유지
