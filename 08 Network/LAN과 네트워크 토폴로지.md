# LAN과 네트워크 토폴로지

## LAN (Local Area Network)
 제한된 지역 내에서 장치들을 연결하는 근거리 네트워크
 - 비교적 짧은 거리
 - 높은 전송 속도
 - 낮은 지연 시간
 - 이더넷(Ethernet) 기술 기반

## 네트워크 토폴로지
 네트워크 장치들이 물리적·논리적으로 연결되는 구조

## 토폴로지 유형
### 버스(Bus) 토폴로지
 > 모든 장치가 하나의 공통 전송 매체에 연결된 구조

 **장점**
 - 구조 단순
 - 설치 비용 낮음

 **단점**
 - 충돌 발생 가능
 - 하나의 회선 장애 시 전체 네트워크 영향
 - 트래픽 증가 시 성능 저하

### 스타(Star) 토폴로지
 > 중앙 장치(허브 또는 스위치)를 중심으로 연결된 구조

 **장점**
 - 관리 및 확장 용이
 - 한 노드 장애 시에도 전체에 영향 없음

 **단점**
 - 중앙 장치 장애 시 전체 네트워크 중단

### 링(Ring) 토폴로지
 > 장치들이 원형으로 연결된 구조   
 > 부분 메시/ 완전 메시 구조 존재

 **장점**
 - 충돌 발생 적음
 - 데이터 전송 순서 예측 가능

 **단점**
 - 하나의 노드 또는 링크 장애 시 전체 네트워크 영향
 - 확장 어려움

### 메시(Mesh) 토폴로지
 > 각 장치가 다수의 장치와 **직접** 연결된 구조

 **장점**
 - 높은 신뢰성
 - 장애 발생 시 우회 경로 존재

 **단점**
 - 구축 비용 높음
 - 구조 복잡
